# Python 算法工程师 — RAG 知识库

## 岗位定义（JD 口径）
「Python 算法」常见有两条主线：（1）**竞赛/工程算法**：用 Python 实现 LeetCode/笔试题、复杂度分析、常见数据结构；（2）**数据与机器学习**：用 NumPy/Pandas/Scikit-learn/PyTorch 等做特征、训练与推理。面试前需确认 JD 偏哪条；本知识库两条都覆盖基础考点。

## 核心技术栈
- Python 语言：列表/字典/堆（heapq）、迭代与生成器、复杂度直觉（append 均摊 O(1) 等）。
- 经典算法：双指针、前缀和、差分、二分、贪心、DFS/BFS、并查集、拓扑排序。
- 动态规划：状态定义、转移、边界、空间优化、DAG 上的 DP。
- 机器学习基础（若相关）：过拟合与正则、交叉验证、偏差方差、常见模型直觉（线性、树、神经网络）。
- 深度学习基础（若相关）：张量、自动微分直觉、优化器、批归一化、过拟合手段。

## 高频考点清单（算法向）
- 复杂度：Big-O 定义、均摊分析、主定理（了解即可）。
- 排序：快排/归并/堆排稳定性与时间空间；何时用内置 sort（Timsort）。
- 图：最短路（Dijkstra/Floyd 适用条件）、最小生成树、二分图判断。
- 字符串：KMP 直觉、哈希滚动（Rabin-Karp 思路）。
- DP：背包、LIS、区间 DP、状态压缩（了解）。

## 高频考点清单（数据科学向）
- NumPy/Pandas：向量化、缺失值处理、groupby、merge、避免 Python 层循环。
- 特征：标准化/归一化、类别特征编码、泄漏（时间泄漏）防范。
- 评估：分类（精确率/召回/F1/AUC）、回归（MAE/RMSE）、不平衡数据策略。

## 优秀回答结构
先陈述**思路**（不变量或子问题），再给出**复杂度**，最后补**边界与反例**。手写代码时先说 API 选择理由（例如用 heapq 维护 TopK）。

## 范例 A：两数之和拓展为「找和为 target 的所有不重复二元组」（算法）
要点范例：排序 + 双指针；跳过重复元素；注意索引与去重条件；时间 O(n log n)。说明为何哈希法在某些去重场景更啰嗦。

## 范例 B：解释过拟合与应对（机器学习）
要点范例：训练好验证差；增多数据、简化模型、正则、早停、特征选择与交叉验证；用学习曲线说明是否数据不足或模型过大。

## 场景题取向
- 数据规模扩大 100 倍：优先考虑算法阶数降低、近似算法、并行与批处理、缓存中间结果。
- 线上模型效果衰减：数据漂移、特征时效、监控与再训练触发条件。

## 项目深挖常见追问
- 你实现的算法模块的输入规模与实测耗时？
- 如何验证正确性（单元测试、对拍、小规模暴力）？
- 若业务要求延迟 < X ms，你如何取舍精度与速度？

## 行为题取向
- 论文/开源与业务交付的时间分配。
- 与工程团队协作：如何将算法落地为可维护服务（批处理、API、监控）。
