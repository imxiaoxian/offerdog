# Web 前端工程师 — RAG 知识库

## 岗位定义（JD 口径）
在浏览器端（含桌面 Web 与部分混合容器）交付交互与可视化：负责组件化 UI、状态与数据流、性能与可访问性、与后端协作的接口契约。技术栈通常覆盖 HTML/CSS、JavaScript/TypeScript、主流框架（Vue/React 等）与构建工具链（Vite/Webpack）。

## 核心技术栈（面试常问范围）
- 基础：DOM 与事件、渲染管线（解析/布局/绘制/合成）、跨域与同源策略、HTTP 缓存。
- JS/TS：事件循环与异步、闭包与内存、模块化、TS 类型系统（收窄、泛型、工具类型）。
- 框架：组件模型、响应式原理（依赖收集/调度）、组合式 API 或 Hooks 心智模型、路由与懒加载。
- 工程化：包管理、代码分割、Tree-shaking、环境变量、Lint/Format、Monorepo 基础。
- 性能：LCP/FCP/CLS/INP、长任务、虚拟列表、图片与字体策略、Service Worker（按需）。
- 质量：单测（组件/逻辑）、E2E 选型、错误监控（Sentry 类）与埋点设计。

## 高频考点清单
- 浏览器：从输入 URL 到首屏发生了什么；重排与重绘；合成层与 GPU。
- 异步：Promise/async、微任务与宏任务顺序、竞态与取消（AbortController）。
- Vue/React：key 的作用、列表 diff 直觉、状态提升、副作用清理。
- 性能：如何定位 LCP 元素、如何减少 JS 体积、如何避免主线程长任务。
- 安全：XSS/CSRF 基础与前端配合点（转义、CSP、Cookie 策略）。

## 优秀回答结构
先给结论，再展开 2～3 个层次：原理 → 在你项目里如何落地 → 如何验证（指标或截图级描述）。避免只罗列 API 名称；用「问题—方案—trade-off」表达选型。

## 范例 A：如何优化首屏（可检索表述）
要点范例：确认 LCP 元素（多为大图或主文案块）；图片用合适格式与尺寸、priority hint、骨架屏；路由级 code split；SSR/SSG 是否值得（看 SEO 与 TTFB）；预连接关键域名；减少阻塞脚本与同步第三方；用 Performance 面板验证长任务与网络瀑布。

## 范例 B：如何处理复杂表单状态
要点范例：拆分字段级校验与提交校验；受控与非受控取舍；用 schema 校验（如 zod+yup 类思路）统一前后端规则；异步校验防抖；错误态与无障碍（aria）；大表单分步与草稿恢复。

## 场景题取向
- 白屏：区分构建错误、运行时异常、资源 404、路由兜底；如何用 source map 与监控定位。
- 内存上涨：Performance 内存快照、Detached DOM、事件未解绑、闭包持有大对象。
- 兼容：渐进增强、特性检测、polyfill 策略与包体权衡。

## 项目深挖常见追问
- 你负责的页面/模块的**性能指标**基线与优化结果？
- 与后端联调中你如何定义类型与错误码（OpenAPI/TS 类型共享等）？
- 一次线上故障的前因后果与你的改进项？

## 行为题取向
- 多需求并行：优先级与沟通节奏。
- 设计稿与实现冲突：如何推动折中并记录决策。
- 代码评审：你关注哪些点（可读性、边界、性能、安全）。
